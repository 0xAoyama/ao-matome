# ジャンル選定 v2 — 短期集客×短期アフィリエイト成果への再構築 2026-06-12

## 結論

3カテゴリ並行（アニメ/AI/生活防衛）をやめ、**「アニメ・推し活」単一バーティカル**に全リソースを集中する。
収益エンジンは以下の2本に絞る。

1. **VOD配信比較（「作品名 + 配信 どこで見れる」）** — 主力
2. **グッズ予約・店舗別特典比較（「作品名 + グッズ 予約 特典」）** — 補助

集客エンジンは「今期アニメ・新作発表・予約開始」のトレンド記事とし、全記事からDBページ（配信比較・特典比較）へ内部リンクで送客する。

## なぜこの2本が「短期間で成果が出る」のか

### VOD配信比較が主力である理由

| 観点 | 評価 |
|---|---|
| CV障壁 | **最小**。無料トライアル登録が成果地点。購入不要 |
| 成果単価 | 高い。VOD系は1件あたり数百〜1,500円程度（ASP案件による） |
| クエリ供給 | 毎クール新作で「○○ 配信 どこ」クエリが新規発生。競合が記事を書く前に出せば新規ドメインでも上位を取れる |
| 在庫切れリスク | なし（物販と違い機会損失がない） |
| 更新性 | 配信開始/終了の差分監視がパイプラインと相性が良い |

対象ASP・案件（要登録・要審査確認）:
- A8.net / もしもアフィリエイト / バリューコマース / afb
- 案件例: U-NEXT、DMM TV、Hulu、ABEMAプレミアム、dアニメストア（時期により募集状況が変わるため登録時に確認）

### グッズ予約・特典比較が補助である理由

| 観点 | 評価 |
|---|---|
| 開始までの速度 | **最速**。楽天アフィリエイトは審査なしで即日リンク発行可。もしも経由でAmazon/Yahooも早い |
| クエリ競合 | 予約解禁直後は競合ゼロに近い。トレンド×購買意図の交点 |
| 単価 | 低い（料率2〜4%）が、件数が積める。サイト全体の「売れる実績」作りに有効 |
| Amazonアソシエイト本体 | 180日以内に3件の売上が必要 → もしも経由を先に使う |

### 撤退するカテゴリとその理由

| カテゴリ | 撤退理由 |
|---|---|
| 生成AIツール | 国内ASPに案件がほぼない。SaaSアフィリは英語圏中心で成果地点が遠い。記事は伸びても収益化導線がない |
| 生活防衛 | クレカ・ポイ活・ふるさと納税は最高単価帯ゆえに超レッドオーシャン。新規ドメインで短期上位は不可能。金融はYMYLで自動公開もできない |
| 美容・コンプレックス（004hで検討） | 単価は最強だがYMYL。E-E-A-T不足の新規ドメインでは成果まで6ヶ月以上。 「短期」要件に反する |

AI・生活防衛の既存記事はv0.0.1タグに残るため削除する（リダイレクトで/newsへ送る）。

## サイト構造 v2

```
/                  トップ: 今期配信表ハイライト + 新着タイムライン + 締切間近の予約
/streaming/        配信どこで見れるDB（一覧）
/streaming/[slug]/ 作品別 配信サービス対応表 ← 主力マネーページ
/goods/            グッズ予約・特典比較DB（一覧）
/goods/[slug]/     商品別 店舗特典比較 ← 補助マネーページ
/news/             ニュース・トレンド記事（集客エンジン）
/news/[slug]/      個別記事 → 末尾で関連DBページへ送客
```

旧URL `/anime/ /ai/ /life/` はリダイレクトする。

## データ設計

- `data/public/streaming.json` — 作品別の配信サービス対応表（サービス名 / 見放題・レンタル / 確認日）
- `data/public/goods.json` — 商品別の店舗・特典・価格比較（店舗 / 特典 / 予約状態 / 締切）
- `site/src/config/affiliates.ts` — サービス別アフィリエイトリンク管理。**ASP承認前はnullにして公式リンクへフォールバック**。承認後にIDを入れるだけで全ページに反映

## デザイン v2（sketches反映）

採用: **004c（アニメ特化・カレンダー帯）× 004g（スケジュール表）× 005（信頼UI）のハイブリッド**

- 白基調 + 紫アクセント（推し活トーン）、ダークは廃止（VOD訴求バナーは白地の方がCTR有利）
- トップに「今期どこで見れる」即答テーブル（004gのカレンダー思想を配信表に転用）
- バッジUI（公式ソース / 確認日 / PR表記）は005から継承 — ASP審査・Adsense審査対策を兼ねる
- 全マネーページにPR表記・確認日・免責を構造として埋め込み
