# 005: Trust Media Hybrid — 本番候補案

## 狙い

これまでの案の「効いた要素」だけを統合した本番候補。白基調の今風メディアUIに青アクセントを乗せ、「公式ソースのみ・転載ゼロ」を構造として見せることで、まとめサイト特有の怪しさを排除しつつ回遊性を保つ。

## 各案から継承したもの

### 002 Editorial Docs から — 信頼感

- 白基調・余白の多いエディトリアルなレイアウト
- トップバーでの信頼ステートメント(「公式ソースのみ」「転載しません」)を常時表示
- ヒーローでサイトの編集方針を一文で宣言する構成

### 001 Precision Dark から — verified/risk UI

- **公開ゲートパネル** — verified(公開可) / risk: review(再確認) / blocked(非公開) の3段階を可視化
- ステータスドット(緑/橙/赤)とモノスペース調のステータス表記
- **ステータスカウンタ** — 公開記事数・監視ソース数・転載ゼロ件数・チェック実施率を数字で提示

### 004 Classic / 004a / 004d から — まとめ回遊導線

- 2カラム構成(メイン + 右サイドバー300px強)
- 右サイドバー: 人気記事ランキング / 公式ソース一覧 / 広告枠(PR表記付き) / 運営ポリシー
- 最新記事一覧のリスト型導線とカテゴリカード
- 004e〜004hで確立したバッジUI(source / risk / checked / PR)

## 必須UIのチェックリスト

- ✅ 白基調の今風メディアUI(#fff ベース、角丸10px、控えめなシャドウ)
- ✅ 青アクセント(#1d4ed8 / #1e3a8a)
- ✅ 公式ソースのみ宣言 + RSS購読CTA(ヘッダー右)
- ✅ verified / risk gate パネル(ヒーロー直下)
- ✅ ステータス: 公開記事数 128 / 監視ソース 42 / 転載 0件 / チェック率 100%
- ✅ AI・ゲーム・美容/コンプレックスの3カテゴリ(美容はYMYLゲート対象を明示)
- ✅ 最新記事一覧(source / risk / checked / PR バッジ付き)
- ✅ 右サイドバー: ランキング / 公式ソース / 広告枠 / 運営ポリシー
- ✅ 本文・画像・SNS転載なしの明記(トップバー + 転載ゼロ宣言ブロック + ポリシー)
- ✅ 自己完結HTML — 外部画像・外部JS・外部フォントなし(サムネイルはCSSグラデーション)

## 強いところ

- **信頼を「言う」のではなく「見せる」** — ゲートパネルとカウンタで編集プロセス自体をUIにしている
- **広告主・読者の両方に説明可能** — PR分離・YMYL対応・転載ゼロが一画面で確認できる
- **まとめサイトの回遊性を犠牲にしない** — ランキング・カテゴリ・最新一覧の導線は004系をそのまま継承
- **白基調でYMYLカテゴリと相性が良い** — 美容・コンプレックス領域でも違和感がない

## 弱いところ

- 2chまとめ的な「賑やかさ」「祭り感」はない — 速報性で勝負する004b/004f系とは別路線
- カウンタとchecked日付は運用が止まると逆効果(古い日付=信頼毀損)になるため自動化が前提
- 情報密度は004e系より低く、1画面あたりの記事露出数では劣る

## 本番採用するなら

- **残す**: 公開ゲートパネル、ステータスカウンタ、バッジUI一式、転載ゼロ宣言、サイドバー構成
- **自動化必須**: カウンタ(CMS集計)、checked日付(公開フロー連携)、ランキング(アクセス解析連携)
- **追加検討**: 記事詳細ページのゲート表示、ソース監視リストの公開ページ化、ダークモード
