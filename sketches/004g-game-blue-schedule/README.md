# 004g-game-blue-schedule

ゲーム特化まとめスケッチ。発売日カレンダーと速報タイムラインの両立を検証する。

## 狙い

ゲーム情報に特化し、「発売日カレンダー」と「速報タイムライン」を1ページに共存させる構成。004b（速報・高密度・赤黒）の情報密度と、004d（青基調・モダン・可読性）のデザインを掛け合わせたバリアント。

## 継承元

### 004b から継承
- 速報ティッカー帯（ヘッダー直下の横スクロール）
- タイムライン形式のニュース一覧（時刻＋カテゴリバッジ＋見出し）
- 高密度レイアウト（1画面に多くの情報を表示）

### 004d から継承
- 青基調のカラースキーム（#2f5fa7）
- 角丸カード・余白の確保
- 読みやすいフォントサイズと行間（line-height: 1.7〜1.8）

## 強いところ

- **発売日カレンダーが実用的**: 月別の発売スケジュールをテーブル形式で一覧でき、プラットフォーム別のカラーバッジ（PS5=青、Switch=赤、PC=灰、Xbox=緑）で対応機種が視覚的にわかる
- **プラットフォーム横断**: 特定ハードに偏らず全機種を並列表示するため、マルチプラットフォームユーザーに刺さる
- **速報と実用情報の共存**: ティッカーで速報性を、カレンダーで実用性を、タイムラインで網羅性をそれぞれ担保
- **コンプライアンスUI完備**: 転載なし・公式ソース要約・PR表記・YMYL注意・確認日・リスクバッジをすべて実装済み

## 弱いところ

- **PV規模が限定的**: ゲーム特化のため総合まとめに比べてリーチが狭い。ゲーム系はレッドオーシャンでもある
- **レビュー/攻略を扱わない方針だと差別化が難しい**: まとめサイトとして発売日情報だけでは公式サイトやファミ通との差別化要素が薄い。ユーザーがここに来る理由を作りにくい
- **発売日データの保守コスト**: カレンダーの情報は手動更新が前提。延期・変更への追従が運用負荷になる
- **インディータイトルの取捨選択**: 掲載基準を決めないと情報量が爆発する

## 本番採用するなら

### 残す要素
- 発売日カレンダー（テーブル形式 + プラットフォームバッジ）
- 速報ティッカー
- タイムライン（カテゴリバッジ付き）
- コンプライアンスバナー一式
- 青基調の配色・角丸カード

### 削る/変更する要素
- カレンダーのデータは静的HTMLではなくJSON/APIから動的生成に変更すべき
- ランキングは外部データソース（Steam売上、ファミ通ランキング等）連携が必要
- コメント欄はモデレーション設計がないと運用できない。実装するなら要設計
- ティッカーのアニメーション速度はユーザー設定可能にすべき（アクセシビリティ）
- 広告枠のサイズ・位置はマネタイズ設計に合わせて調整
